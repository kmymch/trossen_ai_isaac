🤖 Robot Learning Pipeline: Teacher-Student-GRPO Implementation Spec1. Project OverviewImplement a high-performance robotic control pipeline for a WidowX manipulator task in Isaac Lab. The goal is to move from a privileged-state teacher to a vision-based LSTM student policy, finalized with GRPO (Group Relative Policy Optimization).Tech StackSimulator: Isaac Lab (Manager-based Task)RL Library: skrl (v1.1.0+)Environment Management: uvExisting Assets: rsl-rl based WidowX project (to be integrated via inheritance)2. System ArchitectureObservation GroupsGroup policy (Standard): Existing rsl-rl observations.Group teacher (Privileged): Full state (joint pos/vel, object poses, dynamics info).Group student (POMDP): RGB Image + Proprioception (joint pos/vel).Model Architecture (skrl-based)Teacher Model: MLP (Input: privileged_obs) -> Gaussian Policy.Student Model: CNN/ViT Encoder -> LSTM -> Gaussian Policy.Shared Logic: Both must output Gaussian distribution parameters ($\mu, \sigma$). Student must handle rnn_states.3. Implementation PhasesPhase 1: Teacher Training (PPO)Target: Train a privileged MLP policy using skrl.agents.torch.ppo.PPO.Goal: Reach optimal performance in the WidowX task.Output: Save as teacher_policy.pt.Phase 2: DAgger & DistillationMechanism: Student (LSTM) mimics Teacher (MLP) actions.Loop:Rollout using Student Policy.Query Teacher for "Expert Actions" using concurrent privileged_obs.Minimize MSELoss(Student_Action, Teacher_Action).Constraint: Maintain rnn_states consistency during distillation.Phase 3: GRPO Fine-tuningMechanism: Relative evaluation within env groups without a Critic.Agent: Implement GRPOAgent by inheriting skrl.agents.torch.ppo.PPO.Advantage Calculation:Python# Group-relative advantage
mean = grouped_rewards.mean(dim=-1, keepdim=True)
std = grouped_rewards.std(dim=-1, keepdim=True) + 1e-8
advantages = (grouped_rewards - mean) / std
Constraint: Ensure num_envs is a multiple of group_size. Grouped envs must have identical initial states.4.
